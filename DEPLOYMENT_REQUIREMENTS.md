# 神性之旅 AIMS - 部署需求說明

## 📱 應用概述

**應用名稱：** 神性之旅 AIMS - 靈性卡片抽取應用  
**應用描述：** 每日靈性指引，透過 AIMS 四階段探索您的靈魂成長之旅  
**Line@ 帳號：** @360onpll

---

## ✨ 核心功能

### 1. 抽卡功能
- 用戶選擇表情符號（6 種情緒）
- 隨機抽取 360 張靈性卡片（每種情緒 60 張）
- 卡片內容包含：
  - 小靈魂的疑問（藍色文字）
  - 小靈魂問題內容（金色文字）
  - 老靈魂的建議（藍色文字）
  - 老靈魂 AIMS 建議內容（金色文字）
  - AIMS 階段提示（覺察、整合、磁吸、臣服並行動）

### 2. 心情記錄功能
- 用戶可自由選擇表情符號
- 輸入心情日記（文字記錄）
- 自動保存到本地存儲

### 3. 歷史紀錄功能
- 顯示所有過往記錄
- 每筆記錄包含：日期、表情符號、心情內容、卡片信息
- 支持單筆刪除（點擊 × 按鈕確認後刪除）

### 4. 故事頁面
- 展示靈性故事內容
- 點擊「閱讀完整故事」查看詳細內容

### 5. 主頁統計
- 連續打卡天數
- 已抽取卡片數量
- 最後打卡日期
- 歡迎語和靈性提示

### 6. CTA 導向
- 所有頁面底部 LINE@ CTA 按鈕
- 導向 Line@ 帳號：@360onpll

## 🌐 部署需求

### 1. 伺服器要求
- ✅ 靜態文件伺服器（支持 SPA 路由）
- ✅ HTTPS 支持（必須）
- ✅ GZIP 壓縮
- ✅ 跨域資源共享（CORS）

### 2. 路由配置
所有非靜態文件請求應重定向到 `index.html`：
```
/ → index.html
/home → index.html
/history → index.html
/journal → index.html
/stories → index.html
/* → index.html
```

### 3. HTTP 頭配置
```
Cache-Control: public, max-age=31536000, immutable  (靜態資源)
Cache-Control: public, max-age=0, must-revalidate   (HTML)
Content-Type: application/manifest+json             (manifest.json)
X-Content-Type-Options: nosniff
X-Frame-Options: DENY
X-XSS-Protection: 1; mode=block
Referrer-Policy: strict-origin-when-cross-origin
```

### 4. PWA 配置
- ✅ manifest.json 已配置
- ✅ Service Worker (sw.js) 已配置
- ✅ 離線支持已啟用
- ✅ 應用圖標已配置
